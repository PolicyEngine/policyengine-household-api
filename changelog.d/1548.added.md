`/analytics/calculate/requests` now exposes the stored `client_id`: an optional `client_id` query parameter filters both the request list and the `?unique=true` variable summary (served by the existing composite indexes), each serialized request row includes its `client_id`, and the response envelope echoes the filter alongside the other filters. The value is an opaque equality match — the column holds Auth0 M2M client_ids and interactive-user subs alike — and access stays gated behind the existing `read:calculate-analytics` scope, which already guards the whole endpoint.

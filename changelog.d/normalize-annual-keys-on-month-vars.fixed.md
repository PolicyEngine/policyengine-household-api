Match the hosted v1 API when partners send a YEAR-period key on a MONTH-defined variable. Numeric values are treated as annual totals: any explicit monthly inputs are subtracted, and the remainder splits evenly across the unset months. Boolean / string / enum values broadcast unchanged across the unset months while explicit monthly values override. If the explicit monthly inputs sum to more than the annual total, the request is rejected with a 400 (matching policyengine-core's "Inconsistent input" error). Output requests echo the partner's original keys. Adds a `warnings` array to the response when partial monthly input is paired with an annual output for the same year, so partners see a heads-up that the unset months will read the engine's fallback (purely additive — v1 has no such warning, the numeric output is unchanged).

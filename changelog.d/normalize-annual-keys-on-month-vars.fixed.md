When partners send a YEAR-period key on a MONTH-defined variable, the household API now distributes the annual numeric value across the year (V/12 per month) before the engine runs, instead of silently defaulting it to 0 (matches the hosted v1 API for year-only and month-only shapes). Boolean / string / enum values broadcast unchanged. Output requests echo the partner's original keys. When a variable receives both a year key and same-year monthly keys with non-null values, the API resolves the conflict by keeping whichever group appears last in the JSON object and dropping the other (last write wins) — surfacing an `OverlappingPeriodWarning` in the response so partners see exactly what was kept. This last-wins resolution is the household API's own semantic and intentionally diverges from v1's redistribute behavior for the mixed shape.
